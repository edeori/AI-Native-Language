# Development panel — UI

Fájl: [vscode-extension/src/webviews/developmentView.ts](../../vscode-extension/src/webviews/developmentView.ts)

---

## Elrendezés

```
┌─ Context bar ──────────────────────────────┐
│ ✓ semantic  ✓ codegraph  ✗ docs            │
├─ ▶ Last Result ────────────────────────────┤
│ (legutóbb kiválasztott / legfrissebb done) │
├─ ▶ New Task ───────────────────────────────┤
│ [textarea]                                  │
│ [▶ Run Now]  [+ Add to Queue]              │
├─ ▶ Task Log          [▶ Run Queue (N)] ────┤
│ task kártyák fordított időrendben           │
└────────────────────────────────────────────┘
```

Minden szekció összecsukható a fejléc kattintásával. Az állapot csak az aktuális session-re vonatkozik.

---

## Context bar

A panel tetején mindig látható, nem csukható össze.

| Chip | Feltétel |
|---|---|
| `✓ semantic` | `.ai-native/source.semantic.md` létezik |
| `✓ codegraph` | `.ai-native/source.codegraph.json` létezik |
| `✓ docs` | `.ai-native/doc-entities.json` létezik |

---

## No context állapot

Ha sem semantic, sem doc-entities nem létezik, a panel törzsét egy banner váltja fel:

> **No context available.**  
> Run Source Import or Document Import first.  
> `[Open Import Flow]`

Az `Open Import Flow` az `aiNative.openDashboard` commandot hívja.

---

## Last Result szekció

Az utoljára kiválasztott (vagy alapértelmezetten a legfrissebb `done`) task eredményét mutatja.

**Feltöltött állapot:**
```
task-id  [done]  dátum
Összefoglaló szöveg (AI report summary)
📄 N files changed   ⚠ N risks   [view full report]
```

A `view full report` az `aiNative.openImplementationReport` commandot hívja `{ taskId }` payloaddal.  
A szekció frissül ha a Task Log-ban egy taskra kattintunk.

## New Task szekció

### Direction textarea
- Ha Task Log-ban egy kártyára kattintunk, annak direction szövege töltődik be
- Betöltés után a textarea kék kerettel villan és fókuszt kap
- A szöveg szabadon szerkeszthető — beküldve **új taskként** jön létre

### Gombok

| Gomb | Command | Viselkedés |
|---|---|---|
| `▶ Run Now` | `aiNative.runImplementation` | Azonnal elindítja az implementációt `{ direction }` payloaddal |
| `+ Add to Queue` | `aiNative.queueImplementation` | Queue-ba teszi, textarea törlődik |

Ha a textarea üres, mindkét gomb a textarea-ra fókuszál.

## Task Log szekció

### Fejléc

Ha van legalább egy `queued` státuszú task:

```
▶ Task Log          [▶ Run Queue (N)]
```

A `▶ Run Queue (N)` gomb az `aiNative.runQueue` commandot hívja.

### Task kártyák

Fordított időrendben (legfrissebb felül).

```
task-id
direction szöveg (1 sor, ellipsis)    [badge]
dátum                                 [↑ load]   ← hover-re látható
                                      [✕ delete] ← hover-re látható, csak queued státusznál
```

**Státusz badge-ek:**

| Badge | Szín | Jelentés |
|---|---|---|
| `queued` | szürke | Vár a futásra |
| `pending` | kék (halványabb) | Run Queue fut, de ez a task még vár |
| `running` | kék + forgó spinner | Éppen implementálódik |
| `done` | zöld | Befejezett |
| `⚠ drift` | sárga | `docDrift: true` — Claude semantic változást jelzett |

**`✕ delete` gomb** — csak `queued` taskokon látható. Törli a task entryt a `tasks.json`-ból és a `runs/{taskId}/` könyvtárat is.

### Kártyára kattintás

1. Direction szöveg betöltődik a New Task textarea-ba
2. New Task szekció kinyílik
3. Last Result szekció frissül az adott task result-jával

## State model

```typescript
interface DevelopmentViewState {
  hasContext: boolean;
  contextSources: {
    hasSemantic: boolean;
    hasCodegraph: boolean;
    hasDocEntities: boolean;
  };
  tasks: TaskEntry[];
  selectedTaskId?: string;
}
```

### Frissítés

- `DevelopmentWebviewProvider.updateState(partial)` — részleges merge, majd `postMessage`
- `refreshDevelopmentContext()` — artifact fájlok meglétét ellenőrzi, frissíti az összes mezőt
- Minden `refreshViews()` híváskor automatikusan fut

### Webview ↔ Extension kommunikáció

| Irány | `type` | Tartalom |
|---|---|---|
| webview → ext | `ready` | Panel betöltött, kéri az initial state-et |
| ext → webview | `state` | `{ data: DevelopmentViewState }` |
| webview → ext | `command` | `{ command: string, payload: object }` |

---

## Commands

| Command ID | Trigger | Leírás |
|---|---|---|
| `aiNative.runImplementation` | Run Now gomb | Task létrehoz, azonnal futtat |
| `aiNative.queueImplementation` | Add to Queue gomb | Task létrehoz `queued` státusszal |
| `aiNative.runQueue` | Run Queue gomb | Összes queued → sorban futtat |
| `aiNative.deleteTask` | ✕ delete gomb | Queued task törlése (tasks.json + runs mappa) |
| `aiNative.openImplementationReport` | view full report link | Markdown preview a `report.md`-hez |
| `aiNative.openDevelopmentView` | — | Panel fókusz |
| `aiNative.openDashboard` | Open Import Flow gomb | Flow panel fókusz |
